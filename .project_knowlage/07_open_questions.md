# Open Questions

## Zweck dieser Datei
Diese Datei sammelt Themen, die im Repository noch nicht vollständig entschieden oder ausgearbeitet sind. Sie dient als Arbeitsliste für Planner-, Research- und Reviewer-Agenten.

## Aktuelle offene Fragen

### 1. Wie sieht die spätere Referenz-Implementierung konkret aus?
Offen ist, wie stark das Repository später in Richtung Beispielcode, Referenzarchitektur oder Starter-Kit erweitert werden soll.

### 2. Welche Meta-Dateien sollen noch ergänzt werden?
Zum Beispiel:
- ADRs
- Roadmap außerhalb von `.project_knowlage/`
- Indizes
- Review-Checklisten

### 3. Wie tief soll das Thema Governance ausgearbeitet werden?
Der Bereich Datenschutz und Freigaben ist beschrieben, aber eine feinere Governance-Struktur könnte noch ergänzt werden.

### 4. Wie stark soll das Repository auf LangGraph festgelegt werden?
Aktuell ist LangGraph das bevorzugte Orchestrierungsmodell. Offen bleibt, wie neutral oder wie spezifisch die Dokumentation langfristig sein soll.

### 5. Wie sollen spätere Beispielimplementierungen organisiert werden?
Noch offen ist, ob echte Referenzimplementierungen in dieses Repository gehören oder besser in eigene Repositories ausgelagert werden.

## Priorisierung

### Hohe Priorität
- klare Referenzstruktur für Agenten- und Workflow-Beispiele
- ADRs oder Entscheidungsdokumente
- Roadmap für die nächste Ausbaustufe

### Mittlere Priorität
- Governance-Vertiefung
- Meta-Struktur für Indizes und Review-Hilfen
- genauere Trennung zwischen Konzept, Referenz und Implementierung

### Niedrige Priorität
- optische oder kosmetische Verbesserungen
- zusätzliche Beispiele ohne direkten Erkenntnisgewinn

## Arbeitsregel
Offene Fragen sollen sichtbar bleiben, bis sie bewusst entschieden, verschoben oder verworfen wurden.
