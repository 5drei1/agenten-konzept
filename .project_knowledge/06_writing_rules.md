# Writing Rules

## Zweck dieser Datei

Definiert Schreibregeln für alle Dokumente im Projekt, damit Konsistenz gewährleistet ist.

## Sprache

- **Dokumentation**: Deutsch (außer Code-Kommentare)
- **Code**: Englisch (Variablen, Funktionen, Kommentare)
- **Commits**: Englisch
- **Issue-Titel**: Englisch

## Markdown-Konventionen

### Überschriften

- H1 (`#`) nur einmal pro Datei als Titel
- H2 (`##`) für Hauptabschnitte
- H3 (`###`) für Unterabschnitte
- Keine H4+ verwenden (zu tief verschachtelt)

### Listen

- Ungeordnete Listen (`-`) für nicht-sequentielle Inhalte
- Geordnete Listen (`1.`) für sequentielle Schritte
- Maximal 2 Verschachtelungsebenen

### Code-Blöcke

- Immer Sprache angeben: ` ```python `, ` ```json `
- Inline-Code für kurze Ausdrücke: `variable_name`

## Ton und Stil

- **Sachlich und präzise**: Keine blumigen Formulierungen
- **Aktiv statt passiv**: "Agent X führt Y aus" statt "Y wird von Agent X ausgeführt"
- **Konkret**: Beispiele bevorzugen gegenüber abstrakten Beschreibungen
